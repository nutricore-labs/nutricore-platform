# NutriCore Roadmap

NutriCore is developed as a production-oriented health platform for nutrition professionals.

The roadmap is organized around engineering milestones rather than isolated features.

## M1: Platform Foundation

Focus: repository setup, architecture documentation, development workflow, and initial planning.

### Goals

- Define platform architecture
- Define service boundaries
- Establish development workflow
- Create contribution and review guidelines
- Add initial repository quality checks
- Document the product and engineering roadmap

## M2: Identity & Access

Focus: authentication, authorization, and secure access control.

### Goals

- Define authentication flow
- Define role-based access control
- Design JWT and refresh token strategy
- Define service-to-service security direction
- Prepare Auth Service architecture

## M3: Core Domain Workflows

Focus: client and nutrition professional workflows.

### Goals

- Define user profile model
- Define nutrition professional profile model
- Define client management workflows
- Define nutrition plan lifecycle
- Define progress tracking capabilities

## M4: Communication & Scheduling

Focus: secure communication and appointment management.

### Goals

- Define secure messaging workflows
- Define chat service boundaries
- Define appointment scheduling model
- Define calendar integration direction
- Define notification triggers

## M5: Files, Documents & Automation

Focus: file processing, document generation, and archival workflows.

### Goals

- Define secure file upload strategy
- Define file metadata model
- Define document generation workflow
- Define PDF generation approach
- Define archival and retention strategy

## M6: Infrastructure & Observability

Focus: local development, deployment, CI/CD, and monitoring.

### Goals

- Define local Docker environment
- Add CI/CD workflows
- Prepare Kubernetes deployment direction
- Define monitoring and metrics strategy
- Define logging and tracing direction
